## 定位逻辑（仿真默认）

| 问题 | 答案 |
|------|------|
| 用包内 EKF 吗？ | **用。** `localization.launch.py` 启动 EKF +（可选）navsat。 |
| 里程计给 Nav2 | **`/odometry/filtered`**（与 `nav2_params.yaml` 一致）。 |
| GPS / IMU 源 | 默认 **`/roboboat/sensors/gps/navsat`**、**`/roboboat/sensors/imu/imu`**（与 Gazebo 桥一致，可在 launch 改 `gps_src` / `imu_src`）。 |
| `use_sim_time` | Gazebo 运行时 **true**（三处 launch 均建议显式传入）。 |

```text
Gazebo ──► 传感器 ──► covariance repub ──► EKF (+ navsat) ──► /odometry/filtered ──► Nav2
```

## 三终端启动（仿真）

```
终端1 ── Gazebo + bridge
终端2 ── localization（EKF）
终端3 ── Nav2 + RViz
终端4 ── （可选）converter、航点、手测
```

| 步骤 | 命令 |
|:---:|------|
| **1** | `ros2 launch workspace_gz simulation.launch.py` |
| **2** | `ros2 launch workspace_ros localization.launch.py use_sim_time:=true` |
| **3** | `ros2 launch workspace_nav nav2.launch.py use_sim_time:=true` |

默认海图：**`workspace_nav/config/map.yaml`**。换图：

```bash
ros2 launch workspace_nav nav2.launch.py use_sim_time:=true map:=/绝对路径/你的图.yaml
```

## Nav2 → 仿真推进器

仿真 **不要** 使用实船的 MAVROS 速度桥。使用包内 **`converter`**：

```bash
ros2 run workspace_ros converter
```

Nav2 控制器发布 **`/cmd_vel_nav`**（见 `nav2_params.yaml` 与 bringup remap）；`converter` 将其转为 Gazebo 推进器话题。键盘手控可用 **`manual_control`**（见包说明）。

## 航点与地面站

```bash
ros2 run workspace_nav waypoint_transform
ros2 run workspace_nav waypoint_with_state
```

- `waypoint_transform` 默认订 **`/roboboat/sensors/gps/navsat`**；实船接到 USV_NAV 后通常改为 MAVROS GPS 话题。  
- `waypoint_with_state` 默认订 **`/odometry/filtered`**。

## 调试命令速查

```bash
ros2 topic echo /odometry/filtered --once
ros2 topic echo /clock --once
ros2 run tf2_ros tf2_echo map base_link
ros2 topic list | grep roboboat
```

半实物 / 对比：可在 `nav2.launch.py` 设 **`use_mavros_odometry:=true`**，使 Nav2 订阅 **`/mavros/local_position/odom`**（需自行起 MAVROS，细节以 **USV_NAV** 为准）。

## ROS_DOMAIN_ID 与网络

与实船、NX 同网时，用不同 **`ROS_DOMAIN_ID`** 或独立网络，避免话题串线。
