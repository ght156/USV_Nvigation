# 导航接口与参数速查

> 供日常联调、排障、对接时快速查阅。完整字段说明见 [`nav_task_interface.md`](./nav_task_interface.md)。

---

## 1. 话题速查

### 导航发布 → 上层订阅（状态反馈）

| 话题 | 类型 | QoS | 频率 | 说明 |
|------|------|-----|------|------|
| `/nav_status` | `String` (JSON) | RELIABLE + TRANSIENT_LOCAL, depth=10 | 2 Hz | **主通道，必订**。任务状态 + 健康 + 位姿 + 日志 |
| `/task_event` | `String` (JSON) | RELIABLE, depth=50 | 事件驱动 | **强烈建议订阅**。状态跳变 + 告警 |
| `/mission_bridge/state` | `String` | 默认 | 状态变更时 | **回退用**。粗粒度状态值，仅 `/nav_status` 陈旧（>5s）时参考 |

### 上层发布 → 导航订阅（指令下发，兼容保留）

> 推荐使用 Service 接口（§2），Topic 方式保留兼容。

| 话题 | 类型 | 说明 |
|------|------|------|
| `/waypoint` | `String` (JSON) | 下发航线（WGS84 经纬度），fire-and-forget |
| `/color_code` | `String` | 目标颜色，如 `"red"` / `"#FF0000"`，可选 |
| `/gcs_mission/cancel` | `Empty` | 取消任务 / 清除急停 |
| `/gcs_mission/start` | `Empty` | 开始（仅 `start_pulse` 模式） |

## 2. Service 速查

> 上层 → 导航，**推荐**。有同步 ACK（`success`/`message`）。

| Service | 类型 | 请求 | 成功条件 |
|---------|------|------|----------|
| `/mission_bridge/send_waypoints` | `m_common/srv/SendWaypoints` | `PoseStamped[] waypoints` + `string mission_id` | IDLE / COMPLETED / FAILED（RUNNING/PAUSED 需 `allow_replace=true`） |
| `/mission_bridge/set_pause` | `m_common/srv/SetPause` | `bool pause` | `pause=true` 需 RUNNING；`pause=false` 需 PAUSED |
| `/mission_bridge/emergency_stop` | `m_common/srv/EmergencyStop` | 无 | 任意非 EMERGENCY 状态（已 EMERGENCY 时幂等） |

**调用示例**：

```bash
# 下发航线
ros2 service call /mission_bridge/send_waypoints m_common/srv/SendWaypoints \
  "{waypoints: [{header: {frame_id: map}, pose: {position: {x: 10.0, y: 20.0}, orientation: {w: 1.0}}}], mission_id: 'm001'}"

# 暂停
ros2 service call /mission_bridge/set_pause m_common/srv/SetPause "{pause: true}"

# 继续
ros2 service call /mission_bridge/set_pause m_common/srv/SetPause "{pause: false}"

# 急停
ros2 service call /mission_bridge/emergency_stop m_common/srv/EmergencyStop

# 退出急停
ros2 topic pub /gcs_mission/cancel std_msgs/msg/Empty "{}" --once
```

## 3. 状态码速查

### 3.1 `task.state`（7 值）

| 值 | 含义 | 自动恢复？ |
|----|------|-----------|
| `WAITING_SYSTEM` | TF 或 Nav2 action server 未就绪 | TF 就绪后 → IDLE |
| `IDLE` | 空闲，可接收新任务 | — |
| `RUNNING` | 执行中 | — |
| `PAUSED` | 已暂停，保存断点进度 | **否**，需 resume 或 cancel |
| `COMPLETED` | 全部航点到达 | 0.05s 后 → IDLE |
| `FAILED` | 任务失败 | 0.05s 后 → IDLE |
| `EMERGENCY` | 急停，清空一切 | **否**，需 `/gcs_mission/cancel` |

### 3.2 `task_event` 类型

| 事件 | 触发 | 关键字段 |
|------|------|----------|
| `TASK_STARTED` | → RUNNING | `task_id`, `total_waypoints` |
| `TASK_COMPLETED` | → COMPLETED | `task_id`, `elapsed_sec` |
| `TASK_FAILED` | → FAILED | `error_code`, `failed_waypoint_index`, `nav2_error_msg` |
| `TASK_CANCELLED` | RUNNING/PAUSED → IDLE | `task_id`, `source` |
| `TASK_PAUSED` | → PAUSED | `task_id`, `waypoint_index` |
| `TASK_RESUMED` | PAUSED → RUNNING | `task_id`, `waypoint_index` |
| `EMERGENCY_STOP` | → EMERGENCY | `task_id` |
| `ALARM_RAISED` | 告警触发 | `alarm_code`, `level`, `message`, `suggested_action` |
| `ALARM_CLEARED` | 告警解除 | `alarm_code` |

### 3.3 告警码

| `alarm_code` | level | 含义 |
|--------------|-------|------|
| `PLAN_FAILED` | WARN | 全局规划失败 |
| `CTRL_STUCK` | ERROR | 控制器超时无位移 |
| `LOC_LOST` | ERROR | 定位丢失 |
| `LOC_DEGRADED` | WARN | 定位精度下降 |
| `MISSION_FAILED` | ERROR | FollowWaypoints 整体失败 |

## 4. `mission_bridge` 参数速查

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `waypoint_command_mode` | `debounce` | Topic 接口模式：`immediate` / `debounce` / `start_pulse`。**Service 接口无需关心此参数** |
| `waypoint_commit_delay_sec` | `0.45` | debounce 静默窗口（秒） |
| `mission_start_topic` | `""` | `start_pulse` 模式必填 |
| `mission_cancel_topic` | `""` | 取消话题名（如 `/gcs_mission/cancel`）。**同时也是退出 EMERGENCY 的唯一入口** |
| `allow_replace_running_mission` | `false` | 航行中是否允许新任务抢占。影响 Service + Topic |
| `allow_repeat_identical_route` | `false` | 相同 hash 航线完成后是否允许立刻再跑 |
| `suppress_passive_waypoints_after_cancel` | `true` | Cancel 后忽略无 `explicit_replan` 的被动 `/waypoint` |
| `waypoint_tolerance_m` | `1.5` | 航点到达容忍半径（米） |
| `tf_check_period_sec` | `1.0` | TF 就绪检查周期 |
| `target_buoy_force_rewrite` | `false` | 同色是否强制重写 `target_buoy.json` |
| `target_buoy_min_write_period_sec` | `0` | 写 `target_buoy.json` 最短间隔（秒），压限频 |
| `debug_mode` | `false` | 调试日志 |

## 5. `nav_status_aggregator` 参数速查

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `publish_rate` | `2.0` | `/nav_status` 发布频率（Hz） |
| `odom_timeout` | `2.0` | 里程计超时阈值（秒） |
| `gps_timeout` | `5.0` | GPS 超时阈值（秒） |
| `mission_bridge_timeout` | `5.0` | mission_bridge 心跳超时（秒） |
| `cov_lost_threshold` | `10.0` | 协方差超此值判定 LOST |
| `cov_degraded_threshold` | `1.0` | 协方差超此值判定 DEGRADED |
| `stuck_progress_timeout` | `12.0` | 无位移超时判定 STUCK（秒） |
| `vehicle_id` | `usv_001` | 车辆 ID，回显在 `/nav_status` |

## 6. 常用调试命令

```bash
# 查看当前状态
ros2 topic echo /mission_bridge/state --once

# 查看完整 nav_status JSON
ros2 topic echo /nav_status --once

# 确认 TF 链路
ros2 run tf2_ros tf2_echo map base_link

# 确认 FollowWaypoints action server
ros2 action list | grep follow_waypoints

# 查看 Service 列表
ros2 service list | grep mission_bridge

# 查看 Service 接口定义
ros2 interface show m_common/srv/SendWaypoints
ros2 interface show m_common/srv/SetPause
ros2 interface show m_common/srv/EmergencyStop

# 查看 mission_bridge 日志（含状态转换）
ros2 run rqt_console rqt_console  # GUI 过滤 mission_bridge
```

## 7. 状态转换图

```
WAITING_SYSTEM ──(TF+action ready)──► IDLE
IDLE ──(新航点)──► RUNNING
RUNNING ──(全部到达)──► COMPLETED ──(0.05s)──► IDLE
RUNNING ──(航点失败)──► FAILED ──(0.05s)──► IDLE
RUNNING ──(pause)──► PAUSED ──(resume)──► RUNNING
PAUSED  ──(cancel)──► IDLE
PAUSED  ──(新航点)──► RUNNING
任意非EMERGENCY ──(emergency_stop)──► EMERGENCY
EMERGENCY ──(cancel topic)──► IDLE
PAUSED/EMERGENCY ──(TF丢失)──► WAITING_SYSTEM
```
