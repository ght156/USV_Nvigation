## 定位逻辑（实船默认）

| 问题 | 答案 |
|------|------|
| 用包内 EKF 吗？ | **不用。** `real_boat_bringup` 默认 `localization_backend:=mavros_odom`。 |
| 固定地图参考点（datum）？ | **用。** 海图 YAML 的 **`ref_gnss*`**（默认键 **`ref_gnss_10`**），由 **`gnss_odom_map_tf`** 装订 **`map→odom`**。 |
| `navsat.yaml`？ | **模式 B 不启** `navsat_transform`；`datum` 与 `ref_gnss*` **对表**即可。换图以 **map YAML** 为准。 |
| 车体 TF / 里程？ | **MAVROS**：`/mavros/local_position/odom`，**`odom→base_link`**。 |

```text
map YAML ref_gnss* ──► gnss_odom_map_tf ──► map→odom
PX4/MAVROS ──► /mavros/local_position/odom ──► odom→base_link
mission_bridge ──► 同一份 map YAML（map_yaml_path / Nav2 map:=）
```

## 三终端启动（PX4 + MAVROS + Nav2 + Mission）

```
终端1 ── MAVROS
终端2 ── real_boat_bringup（map→odom + 传感器 TF + 速度桥）
终端3 ── nav2_real_mavros（Nav2 + mission_bridge，默认一体）
终端4 ── （可选）地面站 GCS
```

| 步骤 | 命令 |
|:---:|------|
| **1** | `ros2 launch workspace_ros mavros_px4_usv.launch.py fcu_url:=serial:///dev/ttyACM0:57600` |
| **2** | `ros2 launch workspace_ros real_boat_bringup.launch.py use_sim_time:=false enable_nav2_cmd_vel_to_mavros:=true` |
| **3** | `ros2 launch workspace_nav nav2_real_mavros.launch.py use_sim_time:=false` |

终端 3 **默认** `enable_mission_bridge:=true`，会一并启动 `mission_bridge` + `nav_status_aggregator`；**`map_yaml_path` 由 launch 自动与 Nav2 `map:=` 一致**，换图时终端 3 **只改 `map:=`** 即可。

默认 launch 地图：**`workspace_nav/config/map.yaml`**。若现场使用 **`map_real_boat_hk.yaml`**，须 **bringup `map_config_yaml:=` 与 Nav2 `map:=` 指向同一份 yaml**（见「换地图」）。

关闭 mission 栈（回 legacy 三节点）：`enable_mission_bridge:=false`。

**仅测 MAVROS、未起 bringup** 时通常 **没有 `map→odom`**，属正常。

## Nav2 → PX4（速度桥）

Nav2 官方 **`nav2_bringup/launch/navigation_launch.py`** 对速度话题的 remap（与本仓 `nav2_real_mavros.launch.py` 一致）：

```text
controller_server  ──发布──►  /cmd_vel_nav     （控制器原始 Twist）
velocity_smoother    订阅 /cmd_vel_nav，发布 /cmd_vel   （加速度限幅 + 死区后的 Twist）
```

本仓实船速度桥 **默认绕过 smoother**，直接吃控制器输出：

```text
/cmd_vel_nav ──► nav2_cmd_vel_to_mavros ──► /mavros/setpoint_raw/local ──► PX4 OFFBOARD
```

若希望走 **Nav2 平滑后的** `/cmd_vel`，起桥时设 **`cmd_vel_src:=/cmd_vel`**（见下）。

固件约定：`velocity.x` = 前进，`velocity.y` = **偏航角速度**（非侧向速度）。

### 单独启动转接节点（`nav2_cmd_vel_mavros.launch.py`）

MAVROS、**`real_boat_bringup`**（或至少 **`map→odom` + 传感器 TF**）、Nav2 已在其它终端运行时，可**只起速度桥**：

```bash
source /opt/ros/humble/setup.bash
source ~/USV_NAV/install/setup.bash

ros2 launch workspace_ros nav2_cmd_vel_mavros.launch.py use_sim_time:=false
```

| 场景 | 说明 |
|------|------|
| **与 bringup 二选一** | `real_boat_bringup` 已设 **`enable_nav2_cmd_vel_to_mavros:=true`** 时**不要再**单独起本 launch，否则会起两个 `nav2_cmd_vel_to_mavros`。 |
| **台架、不切 OFFBOARD** | 加 **`require_offboard_for_motion:=false`**，便于只看 setpoint。 |
| **走 Nav2 平滑链** | 加 **`cmd_vel_src:=/cmd_vel`**（订阅 smoother 输出，而非 `/cmd_vel_nav`）。 |

常用覆盖示例：

```bash
# 台架：不强制 OFFBOARD 才转发非零速度
ros2 launch workspace_ros nav2_cmd_vel_mavros.launch.py \
  use_sim_time:=false require_offboard_for_motion:=false

# 订阅 velocity_smoother 平滑后的 /cmd_vel（而非默认的 /cmd_vel_nav）
ros2 launch workspace_ros nav2_cmd_vel_mavros.launch.py \
  use_sim_time:=false cmd_vel_src:=/cmd_vel
```

等价单节点（一般不推荐，缺 launch 里整表默认参数时易漏项）：

```bash
ros2 run workspace_ros nav2_cmd_vel_to_mavros --ros-args -p use_sim_time:=false
```

Launch 文件：`workspace_ros/launch/nav2_cmd_vel_mavros.launch.py`；节点实现：`workspace_ros/scripts/nav2_cmd_vel_to_mavros.py`。

**观察**

```bash
ros2 topic echo /cmd_vel_nav
ros2 topic echo /mavros/setpoint_raw/local
ros2 topic echo /mavros/rc/out
```

**OFFBOARD + 解锁**（先起速度桥，再切模式）：

```bash
ros2 service call /mavros/set_mode mavros_msgs/srv/SetMode "{custom_mode: 'OFFBOARD'}"
ros2 service call /mavros/cmd/arming mavros_msgs/srv/CommandBool "{value: true}"
```

**键盘试链**：`teleop_twist_keyboard` 默认发 **`/cmd_vel`**（与 smoother **输出** 同名）。本仓桥默认订 **`/cmd_vel_nav`**，手测请 **` -r cmd_vel:=/cmd_vel_nav`**，或把桥的 **`cmd_vel_src:=/cmd_vel`** 与 teleop 默认话题对齐。

手控不经过桥：`ros2 run workspace_ros mavros_keyboard_teleop` → `/mavros/manual_control/send`。

参数表、保守标定、`nav2_params` 对照：[`实船调试.md`](../src/USV_NAV/docs/实船调试.md)「实船：Nav2 → PX4 OFFBOARD」。

### RPP 前瞻距离 (lookahead) 调参要点

`nav2_params_real_mavros.yaml` 中 **L ≈ clamp(v×lookahead_time, min_lookahead_dist, max_lookahead_dist)**：

| 现象 | 几何原因 | 方向 |
|------|----------|------|
| S 线、左右猛摆、像反向 | **min 过小**，carrot 常在 **最小转弯半径内侧**，船未转完前瞻点已到近处 | **增大 min**（≥ 船头前伸 1.4 m 且 ≥ \(R_{\min}\)）、先 **降速** |
| 目标在船后、先冲很远再拐 | **max 过大**，沿路径切线直行过长 | **减小 max** / `lookahead_time`（勿把 min 收到弯内侧） |

详见本仓 **[`Nav2参数详解与调参指南.md`](./Nav2参数详解与调参指南.md)** §2.2–2.3、§9。

## 调试命令速查（`/cmd_vel_nav` 与 `/cmd_vel`）

### 两个速度话题（Nav2 标准 remap）

| 话题 | 谁发布 | 谁订阅 | 说明 |
|------|--------|--------|------|
| **`/cmd_vel_nav`** | **`controller_server`** | **`velocity_smoother`**；本仓速度桥 **默认**也订此话题 | 控制器 **原始** 速度指令 |
| **`/cmd_vel`** | **`velocity_smoother`** | 下游执行器；teleop 默认名也是 `cmd_vel` | **平滑后** 速度指令 |

Nav2 起好后，导航过程中应 **同时** 能看到两条话题（`hz` 均有数据）：

```bash
ros2 topic list | grep cmd_vel
ros2 topic hz /cmd_vel_nav
ros2 topic hz /cmd_vel
ros2 topic info /cmd_vel_nav -v
ros2 topic info /cmd_vel -v
```

### 速度桥接订哪条

| 目标 | 做法 |
|------|------|
| **默认：绕过 smoother，吃原始指令** | 保持 **`cmd_vel_src:=/cmd_vel_nav`**（launch 默认） |
| **走 Nav2 加速度限幅后的指令** | **`cmd_vel_src:=/cmd_vel`** |

```bash
ros2 launch workspace_ros nav2_cmd_vel_mavros.launch.py \
  use_sim_time:=false cmd_vel_src:=/cmd_vel
```

**不要** 用 `topic_tools relay /cmd_vel /cmd_vel_nav`（方向反了；`/cmd_vel_nav` 应由 controller 发布）。

### 键盘 teleop

- 桥订 **`/cmd_vel_nav`**（默认）时，teleop 应 remap 到 **`/cmd_vel_nav`**：

```bash
sudo apt install -y ros-humble-teleop-twist-keyboard
ros2 run teleop_twist_keyboard teleop_twist_keyboard --ros-args -r cmd_vel:=/cmd_vel_nav
```

- 桥订 **`/cmd_vel`** 时，teleop 可用默认话题名（也是 `/cmd_vel`），注意不要与正在运行的 Nav2 smoother 抢发布。

### 不用 Nav2，手发速度试桥（推荐台架）

发到 **`/cmd_vel_nav`**（与默认桥一致）：

```bash
ros2 topic pub -r 20 /cmd_vel_nav geometry_msgs/msg/Twist "
linear: {x: 0.15, y: 0.0, z: 0.0}
angular: {x: 0.0, y: 0.0, z: 0.1}
"
```

若桥已设 **`cmd_vel_src:=/cmd_vel`**，则把上面话题名改成 **`/cmd_vel`**。松开后应在 **`cmd_timeout_sec`**（默认约 0.3s）内看到 setpoint 变零。

台架不切 OFFBOARD 时：

```bash
ros2 launch workspace_ros nav2_cmd_vel_mavros.launch.py \
  use_sim_time:=false require_offboard_for_motion:=false
```

### 速度链对照（三终端开着时）

```bash
# 输入
ros2 topic echo /cmd_vel_nav
# 或 echo /cmd_vel

# 桥输出
ros2 topic echo /mavros/setpoint_raw/local

# 飞控回读（混控）
ros2 topic echo /mavros/rc/out

# 模式
ros2 topic echo /mavros/state
```

### TF / 定位快速自检

```bash
ros2 run tf2_tools view_frames
ros2 run tf2_ros tf2_echo map base_link
ros2 run tf2_ros tf2_echo odom base_link
ros2 topic echo /mavros/local_position/odom --once
ros2 topic echo /mavros/global_position/global --once
```

### 常用 ROS 命令

```bash
echo $ROS_DOMAIN_ID
ros2 node list
ros2 topic list
ros2 service list | grep -E 'set_mode|arming|set_home'
```

## 地面站与任务（mission_bridge）

地面站（独立仓库 `GROUND-CONTROL-STATION-dev`）通过 ROS Topic 对接；竞赛上层走 Service（见仿真仓 `nav_task_interface.md`）。

**推荐（Nav2 一并启动 mission 栈，默认 `enable_mission_bridge:=true`）**：

```bash
ros2 launch workspace_nav nav2_real_mavros.launch.py use_sim_time:=false
```

等价于 Nav2 + `mission_bridge` + `nav_status_aggregator`。**地图**：`map_yaml_path` 与 Nav2 **`map:=` 自动同源，换图只改 `map:=`**。**mission 参数 YAML**：默认已加载 `mission_stack.real_boat.yaml`（`mission_params_file`），**一般无需手传**。

| Launch 参数 | 默认 | 说明 |
|-------------|------|------|
| `enable_mission_bridge` | `true` | `false` 时不启 mission 栈 |
| `map` | `config/map.yaml` | Nav2 + mission 共用 |
| `mission_params_file` | `mission_stack.real_boat.yaml` | mission 栈专用；**不是** Nav2 的 `params_file`（子 launch 内名为 `mission_stack_params_file`，避免与 Nav2 全局参数冲突） |
| `mission_odom_topic` | `/mavros/local_position/odom` | 与 `nav2_params_real_mavros.yaml` 一致；就绪检查 + 到点容差 + `/nav_status` |
| `params_file` | `nav2_params_real_mavros.yaml` | **仅 Nav2** |

**为何需要地图与里程计**：GCS `/waypoint` 为经纬度 → 须用与 Nav2 同一份 map YAML 的 `ref_gnss*` 转成 map 坐标；odom 未就绪时不发 FollowWaypoints goal。

**仅起 mission 栈（Nav2 已在其它终端运行）**：

```bash
ros2 launch workspace_nav mission_bridge.launch.py use_sim_time:=false
# 地图与 Nav2 不一致时才显式覆盖：
#   map_yaml_path:=/绝对路径/与_Nav2_相同的_map.yaml
```

**勿与下列 legacy 节点并行**（功能已被 `mission_bridge` 覆盖）：

| Legacy | 替代 |
|--------|------|
| `waypoint_transform` + `waypoint_with_state` | `mission_bridge`（`/waypoint` → FollowWaypoints） |
| `target_buoy` | `mission_bridge`（`/color_code` → `target_buoy.json`） |

关闭 mission 栈、继续用旧三节点时：

```bash
ros2 launch workspace_nav nav2_real_mavros.launch.py enable_mission_bridge:=false
# 另开终端
ros2 run workspace_nav waypoint_transform
ros2 run workspace_nav waypoint_with_state
ros2 run workspace_ros target_buoy
```

地面站示例：

```bash
cd <GROUND-CONTROL-STATION-dev>
source venv/bin/activate && npm run start:all
```

联调：`/nav_status.task.state` Dispatch 后 → `RUNNING`；Cancel → `IDLE`。

**迁移 mock 测试**（开发机，不启 MAVROS）：在仿真仓执行  
`bash /path/to/wuxihik_navigation/test/run_usv_migrate_smoke_test.sh`  
说明见仿真仓 `test/MISSION_MIGRATION_TEST.md` 与 [`mission_bridge与上层对接.md`](../src/USV_NAV/docs/mission_bridge与上层对接.md)。

## 换地图

**须同源**：bringup **`map_config_yaml:=`**、Nav2 **`map:=`**（**`enable_mission_bridge:=true` 时 mission 的 `map_yaml_path` 随 Nav2 launch 自动一致**）、PX4 **HOME**（建议对齐 **`map_origin_ref_key`** 角点）。

```bash
ros2 launch workspace_ros real_boat_bringup.launch.py \
  use_sim_time:=false enable_nav2_cmd_vel_to_mavros:=true \
  map_config_yaml:=/绝对路径/你的图.yaml map_origin_ref_key:=ref_gnss_10

ros2 launch workspace_nav nav2_real_mavros.launch.py \
  map:=/绝对路径/你的图.yaml
```

**校验**

- [ ] YAML 含 `ref_gnss_*`，且 **`map_origin_ref_key`** 对应栅格 **(0,0)** 那一角  
- [ ] `gnss_odom_map_tf` 与 `map_server` 为**同一份** YAML  
- [ ] HOME / `set_home` 与所选角点经纬一致（`current_gps: false` 才用指定经纬）  
- [ ] RViz 中船位与栅格无整片平移/拧转  

（仅当将来启用 EKF+`navsat_transform` 时，才必须改 `config/navsat.yaml` 的 `datum`。）

## ROS 网络与域隔离

ROS 2 用 **DDS** 发现同 **`ROS_DOMAIN_ID`** 下的节点。设置 `ROS_LOCALHOST_ONLY=1` 或更换 `ROS_DOMAIN_ID` 后，局域网里其它机器上的节点（例如历史上的 `/usv1/...`）会**看不见**，本机节点仍正常。

**实船 / NX 推荐**（与船载、地面站一致）：

```bash
export ROS_DOMAIN_ID=20
export ROS_LOCALHOST_ONLY=0
export RMW_IMPLEMENTATION=rmw_fastrtps_cpp
```

**本机单独调试**（避免串到办公室其它 ROS）：

```bash
export ROS_DOMAIN_ID=88
export ROS_LOCALHOST_ONLY=1
```

| 变量 | 含义 |
|------|------|
| `ROS_DOMAIN_ID` | 决定能和谁通信 |
| `ROS_LOCALHOST_ONLY` | `1` = 仅本机 |

常用：`ros2 node list`、`ros2 topic list`、`echo $ROS_DOMAIN_ID`。

公司 WiFi 上若多人共用同一 Domain，可能互相发现话题；实船联调宜用**手机热点**或独立 Domain。详见 [`USV 地面站NX 联网部署手册.md`](./USV%20地面站NX%20联网部署手册.md)。
