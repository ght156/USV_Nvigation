# 项目跑通流程与常见调整

本文说明如何从仿真跑到与地面站（GCS）联调，以及如何修改仿真世界的地理原点。依赖安装、工作区编译等请参考仓库内 [`README.md`](../src/YILDIZ-USV/README.md) 的 **DEPENDENCIES** 与 **Step 5–7**。

**扩展阅读**（地图 / GNSS / Nav2 / 仿真是否对齐、仅改 world 是否足够、真北与 `map.yaml` yaw）：[`地图与GNSS-Nav2对齐说明.md`](./地图与GNSS-Nav2对齐说明.md)

---

## 运行前环境

每次开新终端跑 ROS 节点前，先加载 ROS 与**本工作区**（路径按你本机为准，例如 `~/wuxihik_navigation` 或 `~/yildiz_ws`）：

```bash
source /opt/ros/humble/setup.bash
source <你的工作区>/install/setup.bash
```

若使用 **合并安装**（`colcon build --merge-install`），`install` 在 `<工作区>/install/`；若使用**按包隔离安装**，`source` 的仍是 `<工作区>/install/setup.bash`，由 colcon 自动编排各包环境。

---

## 仿真卫生（建议）

避免重复开 Gazebo 导致多世界、桥冲突：

```bash
pkill -f '[g]z sim' 2>/dev/null || true
```

同一时段只保留**一个** `ros2 launch workspace_gz simulation.launch.py`。

---

## 步骤 1～4（与 README Quickstart 一致）

按顺序各占一个终端（或按需在后台运行），**均须已 source 环境与工作区**。

| 步骤 | 说明 | 命令 |
|:---:|:---|:---|
| **1** | 启动仿真 | `ros2 launch workspace_gz simulation.launch.py` |
| **2** | 启动定位 | `ros2 launch workspace_ros localization.launch.py` |
| **3** | 启动 Nav2（默认同时打开 RViz2，配置为 `nav2_bringup` 的 `nav2_default_view.rviz`） | `ros2 launch workspace_nav nav2.launch.py` |
| **4** | 转接（converter） | `ros2 run workspace_ros converter` |

**Nav2 / RViz2**：不要 RViz 时可 `use_rviz:=false`。自定义布局可 `rviz_config:=<你的.rviz全路径>`。仿真下若显示不刷新，在 RViz 左栏 **Global Options** 中勾选 **Use sim time**。

## 地面站（GCS）

在 **[GROUND-CONTROL-STATION](https://github.com/YILDIZ-USV/GROUND-CONTROL-STATION.git)** 项目目录中（路径按你克隆位置而定，例如 `~/GROUND-CONTROL-STATION/`）：

```bash
cd <地面站项目根目录>
source venv/bin/activate
npm run start:all
```

地面站需为后续节点提供**接敌目标信息**与**任务航点经纬度**（见下节步骤 5、6 的说明）。

---

## 步骤 5～7（依赖地面站数据就绪后）

在已 source ROS 与工作区的终端中执行：

| 步骤 | 说明 | 命令 |
|:---:|:---|:---|
| **5** | 目标浮标节点（需 GCS 提供 engagement 目标信息） | `ros2 run workspace_ros target_buoy` |
| **6** | 航点转换（需 GCS 提供任务航点经纬度） | `ros2 run workspace_nav waypoint_transform` |
| **7** | 带状态航点后续流程 | `ros2 run workspace_nav waypoint_with_state` |

以上 5～7 与官方 README 中 **QUICKSTART** 的第 5～7 条一致；差别仅在于：**先启动地面站并完成 `npm run start:all`，再按需启动 5～7**，以便 JSON/话题侧数据由 GCS 注入。

## 调整仿真船「世界原点」经纬度（全球场景）

仿真世界在 Gazebo 的**球坐标**里定义了参考经纬度，与 WGS84 表面对齐。修改下列标签即可把「世界原点」对应到地球上的不同位置，从而配合 NED/经纬度导航链路做各地场景模拟。

**文件位置（源码树）：**

`src/YILDIZ-USV/workspace_gz/worlds/world.sdf`

**修改块示例：**

```xml
<spherical_coordinates>
  <surface_model>EARTH_WGS84</surface_model>
  <world_frame_orientation>ENU</world_frame_orientation>
  <latitude_deg>31.42485</latitude_deg>
  <longitude_deg>120.363668</longitude_deg>
  <elevation>0</elevation>
  <heading_deg>0</heading_deg>
</spherical_coordinates>
```

- **`latitude_deg` / `longitude_deg`**：仿真世界与 WGS84 对齐的参考点（船初始在 world 原点附近时，可与该经纬度一致，便于与 GPS/NavSat、航点转换等链路对照）。
- **`elevation`**：海拔（米）。
- **`heading_deg`**：航向偏置（度），按需要调整。

修改后需**重新启动仿真**（步骤 1）才能生效。

- **仅改经纬度锚点**时：若使用近似「全可通行」栅格图，有时只需同步**任务侧经纬度**即可继续联调；若使用**带岸线或地理参考的地图**，通常还要改 **`map.yaml` 原点/朝向及参考经纬**等，详见 [`地图与GNSS-Nav2对齐说明.md`](./地图与GNSS-Nav2对齐说明.md)。

## 与工作区路径的对应关系

本仓库在磁盘上可能名为 `wuxihik_navigation`，功能包仍位于 `src/YILDIZ-USV/` 下，与上游 YILDIZ-USV 布局一致。文档中的 `ros2 launch …` / `ros2 run …` 包名不变，仅 **`source` 的 `install/setup.bash`** 需指向你本机实际工作区根目录。
