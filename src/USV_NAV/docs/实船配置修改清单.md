# 实船配置修改清单（按文件）

面向：从 **Gazebo 仿真** 迁到 **真 USV（ROS 2 Nav2 + MAVROS/PX4）**。下列按 **文件** 列出要改的 **话题名、参数、launch 变量**；实船**优先**核对 **§1 Nav2** 与 **§2 定位/TF** 中与 **`use_sim_time`、激光、footprint** 相关的行。

**船体尺度 / 质量 / Gazebo 流体与推进插件 / Nav2 footprint / converter 差速系数**：集中说明见 **[`船体与导航参数索引.md`](./船体与导航参数索引.md)**，YAML 索引 **`workspace_nav/config/boat_parameters_index.yaml`（安装后在 `share/workspace_nav/config/`）**。

**两份运行模式（二选一，勿混用 TF/里程）：**

| 模式 | 里程来源 | 说明 |
|------|----------|------|
| **A：本仓库 EKF** | `/odometry/filtered` | 起完整 `localization.launch.py`（IMU/GPS 重发布 + navsat + ekf） |
| **B：MAVROS 飞控融合** | `/mavros/local_position/odom` | `localization_backend:=mavros_odom` + **`ros2 launch workspace_nav nav2_real_mavros.launch.py`**（或旧入口 `nav2.launch.py use_mavros_odometry:=true`） |

## 1. Nav2 与地图（包 `workspace_nav`）

| 文件 | 要改的内容 |
|------|------------|
| **`config/nav2_params_real_mavros.yaml`** | **实船 MAVROS 专用**：全篇 **`use_sim_time: false`**；**`bt_navigator`/`velocity_smoother` 的 `odom_topic`** 为 **`/mavros/local_position/odom`**。**船体 footprint**：1.6×0.8m 保守尺寸，base_link 在船尾前 0.2m（非对称 `[[1.4,0.4],[1.4,-0.4],[-0.2,-0.4],[-0.2,0.4]]`）。**障碍检测**：**`local_costmap`** 使用 **`voxel_layer`（`nav2_costmap_2d::VoxelLayer`, mark_threshold=1）+ `PointCloud2`**，默认话题 **`/livox/lidar`**，膨胀半径 1.2m；**`global_costmap`** 使用 `static_layer` + `voxel_layer` + `inflation_layer`（膨胀半径 1.8m，远距离预判；注意非滚动窗口下动态 ghost 风险）。**控制器**：RPP，前视 1.6~3.5m（min 须 > 船头前伸 1.4m），弯道减速半径 5.0m，代价缩放距离 2.5m。仿真仍用 **`nav2_params.yaml`** 的 **`obstacle_layer` + `LaserScan`**。文件内关键参数均标注中文注释（`——` 标记行）。**换话题 / 改回 2D 激光 / 高度与体素标定**：见 **`实船调试.md`**「避障传感器（模式 B）」与 **注意事项**。 |
| **`config/nav2_params.yaml`** | **仿真默认**：保持不变；仍用 **`/clock`** 时 **`use_sim_time: true`**、**`/odometry/filtered`**。 |
| **`config/nav2_mavros_odom_overlay.yaml`** | 仅当继续用 **`nav2.launch.py` + `use_mavros_odometry:=true`** 时参与合并；若改用 **`nav2_real_mavros.launch.py`**，一般 **不需要** 再合并 overlay。 |
| **`launch/nav2_real_mavros.launch.py`** | **实船推荐**：默认 **`params_file:=nav2_params_real_mavros.yaml`**、**`use_sim_time:=false`**；无运行时 merge，避免和仿真混参。命令示例：`ros2 launch workspace_nav nav2_real_mavros.launch.py` |
| **`launch/nav2.launch.py`**（命令行） | **仿真主入口**（默认 `nav2_params.yaml`、`use_sim_time` 多为 true）。实船若坚持不改文件也可用：**`use_sim_time:=false`**、**`use_mavros_odometry:=true`**、**`params_file:=...`**、**`map:=...`**。更推荐新入口 **`nav2_real_mavros.launch.py`**。 |
| **`config/map.yaml`** + **`map/map.pgm`** | **仿真默认**（`nav2.launch.py`）。**备份**：`config/map.simulation.backup.yaml` + **`map/map.simulation.backup.pgm`**。 |
| **`config/map_real_boat_hk.yaml`** + **`map/hk_map.pgm`** | **实船默认**（`nav2_real_mavros.launch.py`）。园区 HK；**`map:=`** 换图时 **`real_boat_bringup` 须传同一 `map_config_yaml:=`**。 **`ref_gnss_*`** 为 **`[longitude, latitude]`（度）**；**`gnss_odom_map_tf`** 用 **`map_origin_ref_key`** 指定哪一角为 **map 原点**。 |

## 2. 定位与 TF（包 `workspace_ros`）

| 文件 | 要改的内容 |
|------|------------|
| **`config/navsat.yaml`** | **仅模式 A**。**`datum`**：`[lat, lon, heading_rad]`，须与 **`map.yaml`**（仿真）或实船 **`map_server`** 所用地图的 **`ref_gnss*`** 一致；实船默认图为 **`map_real_boat_hk.yaml`**。**`frequency`**、**`delay`** 可按现场调。 |
| **`config/ekf.yaml`** | **仅模式 A**。 **`imu0`**：默认 **`/imu/fixed_cov`**（来自 `imu_covariance_repub`）。 **`odom0`**：默认 **`/odometry/gps`**（navsat_transform 输出）。若重映射 IMU/GPS 链，改这里或改 repub 输出名。 **`world_frame`/`map_frame`/`odom_frame`**：一般勿动除非你清楚整树。 |
| **`config/static_transform.yaml`** | **仿真 / `localization.launch.py`（模式 A）** 沿用：与 Gazebo 命名（`*_link`→`usv/base_link/sensor_*`）一致；为保持与原仿真 README 一致，**请保留此文件语义供仿真链路使用**。 |
| **`config/static_transform_real_boat.yaml`** | **实船模式 B**（`localization_backend:=mavros_odom`、`real_boat_mavros_tf` / 旧名 `real_boat_tf_static`）：**`base_link`→`lidar_link`/IMU/GPS/相机**；平移默认同 **`usv.xacro`**，真机按标定改。另可 **`static_transform_file:=<路径>`** 覆盖。 |
| **`launch/localization.launch.py`**（命令行） | **`use_sim_time:=false`**。 |
| **`launch/real_boat_bringup.launch.py`**（命令行） | **`use_sim_time:=false`**。**`localization_backend`**：**`robot_localization`**（模式 A，`static_transform.yaml`）或 **`mavros_odom`**（模式 B，默认 **`static_transform_real_boat.yaml`**，可 **`static_transform_file:=...`**）。**模式 B：`map_config_yaml`、`map_origin_ref_key`** — 传给 **`real_boat_mavros_tf` / `gnss_odom_map_tf`**：须与 Nav2 **`map:=`** **同一路径**；键名表示 **栅格 (0,0)** 对应 **`ref_gnss*`** 哪一角（默认 **`ref_gnss_10`**）。另 **`map_odom_yaw_deg`**、**`scan_src`**。速度桥二选一（**勿同时 true**，launch 会报错退出）：**`enable_nav2_cmd_vel_to_mavros`**（推荐，`/cmd_vel`→`/mavros/setpoint_velocity/cmd_vel_unstamped`，见 **`nav2_cmd_vel_to_mavros`**）或 **`enable_nav2_to_mavros_cmd_vel`**（legacy，`/cmd_vel_nav`→setpoint_velocity）。 |
| **`launch/real_boat_mavros_tf.launch.py`**（薄封装旧名 **`real_boat_tf_static`**） | 直接起静态传感器 TF + **`gnss_odom_map_tf`** 或静态 **`map→odom`**；参数与 bringup 一致，见 **[`实船调试.md`](./实船调试.md)**「`gnss_odom_map_tf` 与地图锚点」。 |
| **`scripts/gnss_odom_map_tf.py`** | 依赖 **`python3-yaml`**（`package.xml` **`exec_depend`**）。读 **`map_config_yaml`** 中 **`map_origin_ref_key`**；空路径时退化为手工 **`map_origin_latitude/longitude`**。**平移计算公式、未实现项、`initialize_once` 行为**：见 **`实船调试.md`** → **「算法与实现要点（复盘／改代码时从这里对）」**。 |

## 3. MAVROS 桥接与传感器话题名（包 `workspace_ros`）

| 文件 | 要改的内容 |
|------|------------|
| **`launch/mavros_px4_usv.launch.py`** | **实船推荐**（模式 B 先起）：替代裸 **`ros2 launch mavros px4.launch`**。默认链：**已安装** **`px4_pluginlists.yaml`** → **`px4_config.yaml`** → **`config/mavros_px4_overrides_usv.yaml`**；**`fcu_url:=/dev/ttyACM0:57600`** 等按机载改。需要换路径时传 **`pluginlists_yaml`** / **`base_config_yaml`** / **`override_config_yaml`**。 |
| **`config/mavros_px4_overrides_usv.yaml`** | **只改与 TF/帧名相关的少数键**（`local_position`、`global_position`、`odometry` 等），与 Nav2 **`map`→`odom`→`base_link`** 一致；勿在仓库里维护整份上游 **`px4_config.yaml`**。 |
| ~~`scripts/mavros_usv_relay.py`~~ | **已移除**。模式 A 的 IMU/GPS 数据由 MAVROS 直连 **`imu_covariance_repub`** / **`gps_covariance_repub`**（通过 `imu_src`/`gps_src` 参数指定源话题）。模式 B 不再经过 `/usv` 命名空间。 |
| **`scripts/nav2_cmd_vel_to_mavros.py`** + **`launch/nav2_cmd_vel_mavros.launch.py`** | **推荐实船 OFFBOARD 链**：订阅 **`/cmd_vel`**，发布 **`/mavros/setpoint_velocity/cmd_vel_unstamped`**；参数含限幅、死区、超时、`/mavros/state` **OFFBOARD 门控**、`offboard_cmode_allowlist`、`forbid_reverse`、`min_surge_for_turn` 等。启动：`real_boat_bringup` **`enable_nav2_cmd_vel_to_mavros:=true`** 或单独 **`ros2 launch workspace_ros nav2_cmd_vel_mavros.launch.py`**。**调试**：**`ros2 topic echo /mavros/setpoint_velocity/cmd_vel_unstamped`**、**`/mavros/rc/out`** —— **`实船调试.md`**「**MAVROS 侧观察（调试）**」。 |
| **`scripts/imu_covariance_repub.py`** | 订阅话题：通过参数 **`imu_topic`** 指定（默认 **`/usv/sensors/imu/imu`**）；实船模式 A 由 **`real_boat_bringup`** 的 **`imu_src`** 传入 MAVROS 话题（如 **`/mavros/imu/data`**）。 |
| **`scripts/gps_covariance_repub.py`** | 订阅：通过参数 **`gps_topic`** 指定（默认 **`/usv/sensors/gps/navsat`**）；实船模式 A 由 **`real_boat_bringup`** 的 **`gps_src`** 传入 MAVROS 话题（如 **`/mavros/global_position/raw/fix`**）。 |
| **`scripts/converter.py`** | **真船若不走 Gazebo 推力 topic**：通常 **停掉** 本节点，或改 **`left_thruster_topic`/`right_thruster_topic`**、**`linear_scale`/`angular_scale`** 对接你方电控；**与 MAVROS cmd_vel 二选一**，避免双通道抢控。 |

## 6. 实船最小启动时“要改的变量”速记（命令行）

```text
# MAVROS（终端1，示例）：推荐本包 launch + 覆盖层
ros2 launch workspace_ros mavros_px4_usv.launch.py fcu_url:=设备   # 如 /dev/ttyACM0:57600

# real_boat_bringup（终端2）— 打开推荐速度桥时示例：
use_sim_time:=false
localization_backend:=mavros_odom   # 或 robot_localization
scan_src:=/真雷达话题                # 可选
enable_nav2_cmd_vel_to_mavros:=true # 推荐：/cmd_vel → setpoint_velocity（勿与 enable_nav2_to_mavros_cmd_vel 同时 true）
# 与 Nav2 使用同一张图（换图时两处路径一致）：
map_config_yaml:=<与 map_server 相同的 map.yaml 绝对路径>
map_origin_ref_key:=ref_gnss_10     # 或 ref_gnss_00 等，须与栅格 (0,0) 角点一致

# Nav2 — 推荐独立 launch + 独立参数文件（不影响仿真默认）
ros2 launch workspace_nav nav2_real_mavros.launch.py map:=<同上 map.yaml>
# 等价于默认 params_file:=.../nav2_params_real_mavros.yaml ，use_sim_time:=false

# 仅启速度桥（bringup 未带 enable_nav2_cmd_vel_to_mavros 时，终端可选）：
# ros2 launch workspace_ros nav2_cmd_vel_mavros.launch.py use_sim_time:=false
# 联调观察：ros2 topic echo /mavros/setpoint_velocity/cmd_vel_unstamped
#           ros2 topic echo /mavros/rc/out

# 若仍用旧入口 nav2.launch.py + 仿真 nav2_params.yaml：
# ros2 launch workspace_nav nav2.launch.py use_sim_time:=false use_mavros_odometry:=true
```

**联调下一项（速度指令 / OFFBOARD）**：**`nav2_cmd_vel_to_mavros`**、**`ros2 topic echo /mavros/setpoint_velocity/cmd_vel_unstamped`**、**`/mavros/rc/out`**、**`nav2_params_real_mavros.yaml`** —— **`实船调试.md`**「**实船：Nav2 `/cmd_vel` → PX4 OFFBOARD**」。
