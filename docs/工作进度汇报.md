## 仿真 vs 实船（汇报用一页纸）

| 维度 | **仿真（本仓库）** | **实船（USV_NAV）** |
|------|-------------------|---------------------|
| **目标** | Gazebo 动力学、Nav2、EKF、地面站链路的**研发与回归验证** | 现场 **MAVROS / PX4 / NX**、传感器、**map→odom** 装订、实航 |
| **定位源** | `localization.launch.py` → **`/odometry/filtered`**；`navsat.yaml` **固定 datum**（**不自动读海图**，须与 `ref_gnss*` 手调一致） | **`/mavros/local_position/odom`** + **`gnss_odom_map_tf`** 等（以 USV_NAV 文档为准） |
| **默认海图** | 当前工作流默认 **`map_hk.yaml`** + 对应 PGM（可按项目切 `map:=`） | 实船海图与 HK/现场图在 **USV_NAV** 维护 |
| **地面站 /waypoint** | **`mission_bridge`**（可选 `enable_mission_bridge:=true`）、**`waypoint_transform`**；载荷格式与 USV 侧对齐 | **`mission_bridge`** 等在现场 launch 中集成（开发主线在 USV_NAV） |
| **推进** | **`converter`**（`/cmd_vel_nav` → Gazebo 推力） | **速度桥 / PX4 OFFBOARD**（USV_NAV） |
| **维护策略** | 去掉双份 MAVROS 实船栈，**避免与 USV_NAV 同步两套** | 实船独占配置与 launch **仅在此仓演进** |

### 关键结论（可写进报告）

1. **双栈分工已明确**：仿真专注可重复验证；实船专注现场与飞控链，减少「改一处要改两处」的维护成本。  
2. **navsat `datum` 与海图 `ref_gnss*` 必须同角点**，但 **robot_localization 不会自动读 map yaml**，换图需同步改 `navsat.yaml`。  
3. **地面站在航迹执行中改目标**：`mission_bridge` 可提供 **cancel 当前 FollowWaypoints** 再派新任务的路径（参数 **`allow_replace_running_mission`**，launch **默认 false**（实船安全）；仿真需在运动中自动换路时显式 **`true`**，约 **0.22s** 延迟降低抢占竞态）。**Cancel Nav 后**默认 **`suppress_passive_waypoints_after_cancel:=true`**，避免 GCS **被动 `/waypoint`**（无 **`explicit_replan`**）紧接着再次 `_execute_mission`；要继续飞须 **Run Mission 首帧显式重申**或 **`start_pulse` + Empty**。
4. **订阅地面站话题**（`/waypoint`、`/color_code`）为轻量回调，**不是**系统卡顿主因；性能热点在仿真与 Nav2 规划/代价图。**`target_buoy`**：**同色不写**降噪；仍需限频时对 **`target_buoy_min_write_period_sec`** 调参。
