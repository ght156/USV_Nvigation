## 运行前环境

每次开新终端跑 ROS 节点前，先加载 ROS 与**本工作区**（路径按你本机为准，例如 `~/wuxihik_navigation` 或 `~/usv_nav_ws`）：

```bash
source /opt/ros/humble/setup.bash
source <你的工作区>/install/setup.bash
```

若使用 **合并安装**（`colcon build --merge-install`），`install` 在 `<工作区>/install/`；若使用**按包隔离安装**，`source` 的仍是 `<工作区>/install/setup.bash`，由 colcon 自动编排各包环境。

## 实船：整体 launch 路线与启动命令（PX4 + MAVROS）

下列为 **从零到 Nav2** 的推荐顺序；**每个终端前先** `source /opt/ros/humble/setup.bash` **与** `source <工作区>/install/setup.bash`。

```
终端1 ──► MAVROS（飞控）
终端2 ──► real_boat_bringup（传感器桥 / map→odom；建议打开 Nav2→MAVROS 速度桥）
终端3 ──► Nav2（实船参数 `nav2_real_mavros.launch.py`）
终端4 ──► （可选）键盘发 /cmd_vel 试链：teleop_twist_keyboard；或地面站 / 航点 5～7
```

**仿真用 `converter`（`/cmd_vel_nav`→Gazebo 推力）与实船 PX4 不是同一条链**；实船 OFFBOARD 速度链见下节 **「Nav2 `/cmd_vel` → PX4：速度转换节点」**。

### 模式 B：飞控融合里程 + Nav2（推荐先试通）

**含义**：Nav2 里程话题用 **`/mavros/local_position/odom`**；**不**起包内 **`ekf_node` / `navsat_transform`**；**`map→odom`** 默认由 **`gnss_odom_map_tf`** 根据 **GNSS + 局域 odom** 装订（可选恒等静态：`use_gnss_map_odom_tf:=false`）。**地图锚点**须与 **`map_server` 使用同一份 YAML**，见 [`实船调试.md`](../src/USV_NAV/docs/实船调试.md)「`gnss_odom_map_tf` 与地图锚点」。

| 步骤 | 终端 | 命令 |
|:---:|:---|:---|
| **1** | MAVROS | `ros2 launch workspace_ros mavros_px4_usv.launch.py fcu_url:=serial:///dev/ttyACM0:57600` |
| **2** | 实船 bringup | `ros2 launch workspace_ros real_boat_bringup.launch.py use_sim_time:=false localization_backend:=mavros_odom enable_nav2_cmd_vel_to_mavros:=true`（**务必**与下一步 Nav2 使用 **同一** **`map_config_yaml`**；换图示例见 **`实船调试.md`**） |
| **3** | Nav2（实船参数；**默认 `map.yaml` + `map/map.pgm`**） | `ros2 launch workspace_nav nav2_real_mavros.launch.py`（换图：`map:=/路径/其它.yaml`，且 bringup **`map_config_yaml`** 对齐同路径） |

**联调闭环（推进）**：步骤 2 若已加 **`enable_nav2_cmd_vel_to_mavros:=true`**，则 **Nav2 的 `/cmd_vel`** 会经 **`nav2_cmd_vel_to_mavros`** 转到 **`/mavros/setpoint_velocity/cmd_vel_unstamped`**（PX4 **OFFBOARD**）。详见下文 **「Nav2 `/cmd_vel` → PX4：速度转换节点」**；更细的命令、`ros2 topic echo` 观察（含 **`/mavros/rc/out`**）、参数表与手动测试见 [`实船调试.md`](../src/USV_NAV/docs/实船调试.md)。

- **`fcu_url`**：按本机串口改（如 `/dev/ttyACM1`）。  
- **避障传感器**：**默认实船 Nav2** 用 **`nav2_params_real_mavros.yaml`** 的 **`VoxelLayer` + `PointCloud2`**（如 **`/livox/lidar`**），并在该 yaml 中改 **`voxel_layer.livox_cloud.topic`** 与 TF（详见 **`实船调试.md`**「避障传感器」）。
- **`enable_nav2_cmd_vel_to_mavros`**：`true` 时随 bringup 启动 **`nav2_cmd_vel_to_mavros`**（**/cmd_vel**→**setpoint_velocity**）。也可在 **终端4** 单独：`ros2 launch workspace_ros nav2_cmd_vel_mavros.launch.py use_sim_time:=false`。完整命令、**`nav2_params_real_mavros.yaml` 保守项**、**`ros2 topic echo /mavros/setpoint_velocity/cmd_vel_unstamped`** 与 **`/mavros/rc/out`** 见 **[`实船调试.md`](../src/USV_NAV/docs/实船调试.md)**「**实船：Nav2 `/cmd_vel` → PX4 OFFBOARD**」→「**MAVROS 侧观察（调试）**」。  
- **传感器静态 TF**：默认 **`workspace_ros/config/static_transform_real_boat.yaml`**（**`base_link`→各传感器 link**）；仿真的 **`localization.launch`** 仍用 **`static_transform.yaml`**。也可用 **`static_transform_file:=<YAML 绝对或 share 路径>`** 指向自定义副本。  
- **PX4 HOME 与海图**：联调 RViz **`map`** 前，建议将 HOME 对齐到 **`nav2_real_mavros`** 当前载入 YAML（默认 **`workspace_nav/config/map.yaml`**）中的约定 **`ref_gnss*`**；可用 ROS 2 **`ros2 service call /mavros/cmd/set_home mavros_msgs/srv/CommandHome ...`**（注意 **`current_gps: false` 才是指定经纬度**）。可复制示例与自检见 **[`实船调试.md`](../src/USV_NAV/docs/实船调试.md)** §「用 ROS 2 服务设置 PX4 HOME」及「`map`/odom/HOME」。  
- **PX4 真机推进**：一般**不要**再跑仿真用 **`converter`**。  
- **仅测 MAVROS、未起 bringup**：树里不会有 **`map→odom`**，可临时：`ros2 run tf2_ros static_transform_publisher 0 0 0 0 0 0 map odom`。

### Nav2 `/cmd_vel` → PX4：速度转换节点（实船联调必备项）

**作用**：把 Nav2 **`velocity_smoother`** 输出的 **`geometry_msgs/Twist`**（**`/cmd_vel`**）接到 **MAVROS** **`/mavros/setpoint_velocity/cmd_vel_unstamped`**，供 **PX4 OFFBOARD** 使用。**不是**仿真里的 **`converter`**（**`/cmd_vel_nav`→`/usv/thrusters/...`** → Gazebo）。

**推荐控制链**：

```text
Nav2 → /cmd_vel → nav2_cmd_vel_to_mavros → /mavros/setpoint_velocity/cmd_vel_unstamped → PX4
```

**怎么起**（二选一）：

1. **随 `real_boat_bringup`**（推荐，与上表步骤 2 一致）：  
   `... real_boat_bringup.launch.py ... enable_nav2_cmd_vel_to_mavros:=true`  
2. **单独终端**（bringup 未带该开关时）：  
   `ros2 launch workspace_ros nav2_cmd_vel_mavros.launch.py use_sim_time:=false`

**观察链路（建议另开终端）**：

```bash
ros2 topic echo /cmd_vel
ros2 topic echo /mavros/setpoint_velocity/cmd_vel_unstamped
ros2 topic echo /mavros/rc/out
```

**`/mavros/setpoint_velocity/cmd_vel_unstamped`**：桥写入的速度设定；**`/mavros/rc/out`**：飞控侧经 MAVROS 回读的 **RC 输出**（通道 PWM 等），便于对照 **指令 → 混控输出** 的时延与饱和（类型以 `ros2 topic info` 为准）。

**PX4：切 OFFBOARD + 解锁（可复制）**

建议顺序：**先起 `nav2_cmd_vel_to_mavros`**（默认 **~20 Hz 持续发零速** setpoint，满足 PX4 「先收到流再切 OFFBOARD」的常见要求），再切模式、再 arm（以队里安全规范为准）。

```bash
ros2 service call /mavros/set_mode mavros_msgs/srv/SetMode "{custom_mode: 'OFFBOARD'}"
ros2 service call /mavros/cmd/arming mavros_msgs/srv/CommandBool "{value: true}"
ros2 topic echo /mavros/state --once
```

`require_offboard_for_motion:=true`（默认）时，**非 OFFBOARD** 会拦截非零 `/cmd_vel`；台架只测话题链时可 **`require_offboard_for_motion:=false`**。

**不用 Nav2、先键盘试链**（仍经过同一转换节点）：系统包 **`ros-humble-teleop-twist-keyboard`** 往 **`/cmd_vel`** 发 Twist：

```bash
sudo apt install ros-humble-teleop-twist-keyboard
ros2 run teleop_twist_keyboard teleop_twist_keyboard
```

**与 `mavros_keyboard_teleop` 的区别**：后者发 **`/mavros/manual_control/send`**，**不经过** `nav2_cmd_vel_to_mavros`。

**参数与保守标定**：[`实船调试.md`](../src/USV_NAV/docs/实船调试.md)「实船：Nav2 `/cmd_vel` → PX4 OFFBOARD」；按文件修改索引：[`实船配置修改清单.md`](../src/USV_NAV/docs/实船配置修改清单.md) §3。

### 模式 A：包内 EKF + navsat（与仿真定位链一致）

**含义**：`real_boat_bringup` **`localization_backend` 默认 `robot_localization`**，会起 **`localization.launch.py`**（IMU/GPS repub、`navsat_transform`、`ekf_node`），末尾同样有 **恒等 `map→odom`**；传感器静态 TF 仍读仿真用 **`static_transform.yaml`**；Nav2 订阅 **`/odometry/filtered`**。

| 步骤 | 终端 | 命令 |
|:---:|:---|:---|
| **1** | MAVROS（向 `/usv` 供电调话题时需先起） | `ros2 launch workspace_ros mavros_px4_usv.launch.py fcu_url:=/dev/ttyACM0:57600` |
| **2** | 实船 bringup | `ros2 launch workspace_ros real_boat_bringup.launch.py use_sim_time:=false enable_nav2_cmd_vel_to_mavros:=true`（若仍用 MAVROS 里程推进 PX4，速度与 **模式 B** 相同；**localization_backend** 默认 **EKF**，勿与 MAVROS 重复发 **`odom`→`base_link` TF**，见下节警告） |
| **3** | Nav2 | `ros2 launch workspace_nav nav2.launch.py use_sim_time:=false` |

**TF 警告**：模式 A 下 **`ekf_node` 会发 `odom→base_link`**。若 MAVROS 仍按 **`mavros_px4_overrides_usv.yaml` 发送同一 TF**，会与 EKF **重复** ——须在飞控调试阶段 **关闭 MAVROS 侧 `local_position` TF** 或换用不提供车体 TF 的 MAVROS 配置，详见 [`实船调试.md`](../src/USV_NAV/docs/实船调试.md)「模式 A / B」。

### TF 复盘清单（实船）

- **目标**：**`map`→`odom`→`base_link`→`lidar_link`**（或与 **`LaserScan.frame_id`** 一致）全流程不断。  
- **模式 B**：**`map`→`odom`** → **`real_boat_mavros_tf.launch.py`**（旧名 **`real_boat_tf_static.launch.py`**），默认 **`gnss_odom_map_tf`**（**`map_config_yaml`** / **`map_origin_ref_key`**，与 Nav2 **`map:=`** 同源）；关 **`use_gnss_map_odom_tf`** 时静态 **`tf2_ros`**。**`odom`→`base_link`** → **`mavros_px4_usv`** + **`mavros_px4_overrides_usv.yaml`**；传感器 → **`static_transform_real_boat.yaml`**（可 **`static_transform_file:=...`**）。  
- **命令**：**`ros2 run tf2_tools view_frames`**、**`tf2_echo map base_link`**、核对 **`local_position/odom`** 的 **`frame_id`**；全文见 **`实船调试.md`**「TF 坐标系总览与复盘调试」。  

### 实船可选用节点（与地面站）

与仿真 **步骤 5～7** 相同：地面站就绪后按需 **`target_buoy` / `waypoint_transform` / `waypoint_with_state`**。

- **键盘 → `/mavros/manual_control/send`（手控链路）**：`ros2 run workspace_ros mavros_keyboard_teleop`（**不**经过 **`nav2_cmd_vel_to_mavros`**）。  
- **键盘 → `/cmd_vel`（与 Nav2 相同，可测速度桥）**：`ros2 run teleop_twist_keyboard teleop_twist_keyboard`（需已 **`apt install ros-humble-teleop-twist-keyboard`**）。详见上文 **「Nav2 `/cmd_vel` → PX4：速度转换节点」**。

## 调整仿真船「世界原点」经纬度（全球场景）

仿真世界在 Gazebo 的**球坐标**里定义了参考经纬度，与 WGS84 表面对齐。修改下列标签即可把「世界原点」对应到地球上的不同位置，从而配合 NED/经纬度导航链路做各地场景模拟。

**文件位置（源码树）：**

`src/USV_NAV/workspace_gz/worlds/world.sdf`

**修改块示例：**

```xml
<spherical_coordinates>
  <surface_model>EARTH_WGS84</surface_model>
  <world_frame_orientation>ENU</world_frame_orientation>
  <latitude_deg>31.42485</latitude_deg>
  <longitude_deg>120.363668</longitude_deg>
  <elevation>0</elevation>
  <heading_deg>0</heading_deg>
</spherical_coordinates>
```

- **`latitude_deg` / `longitude_deg`**：仿真世界与 WGS84 对齐的参考点（船初始在 world 原点附近时，可与该经纬度一致，便于与 GPS/NavSat、航点转换等链路对照）。
- **`elevation`**：海拔（米）。
- **`heading_deg`**：航向偏置（度），按需要调整。

修改后需**重新启动仿真**（步骤 1）才能生效。

- **仅改经纬度锚点**时：若使用近似「全可通行」栅格图，有时只需同步**任务侧经纬度**即可继续联调；若使用**带岸线或地理参考的地图**，通常还要改 **`map.yaml` 原点/朝向及参考经纬**等，详见 [`地图与GNSS-Nav2对齐说明.md`](./地图与GNSS-Nav2对齐说明.md)。

## 更换地图时需要修改的参数

最简单的方式就是替换地图的名字和配置文件的名字。

当切换到不同的栅格地图（例如从 `map.yaml` 换到另一个区域的 PGM + YAML）时，以下 **4 个位置必须同步更新**，否则 GNSS 定位、航点转换、map→odom 变换会使用错误的地图锚点。

### 必须同步修改的 4 项

| # | 文件 | 参数 | 说明 |
|:--:|------|------|------|
| 1 | `workspace_ros/config/navsat.yaml` | `datum: [lat, lon, heading]` | **EKF navsat_transform 的局部平面原点**，必须与地图 YAML 中 `ref_gnss_10`（左下角）一致 |
| 2 | `workspace_ros/launch/real_boat_bringup.launch.py` | `default_map_yaml`（第 51-52 行） | 传给 `gnss_odom_map_tf` 的 `map_config_yaml` 默认路径，用于动态 map→odom 变换 |
| 3 | `workspace_ros/launch/real_boat_mavros_tf.launch.py` | `default_map_yaml`（第 39-40 行） | 同上，直接启动 mavros_tf 时的默认地图路径 |
| 4 | `workspace_nav/workspace_nav/waypoint_transform.py` | fallback 路径（第 185 行） | 当 `map_yaml_path` 参数为空时，默认读取的地图 YAML 路径 |

### 地图 YAML 中需要的字段

```yaml
image: ../map/your_map.pgm
resolution: 1.0
origin: [0.0, 0.0, 0.0]         # 左下角在 map 系的位姿
ref_gnss: [lon_center, lat_center]   # 地图中心经纬度（参考用）
ref_gnss_00: [lon_tl, lat_tl]       # 左上角
ref_gnss_01: [lon_tr, lat_tr]       # 右上角
ref_gnss_10: [lon_bl, lat_bl]       # 左下角 ← 作为 datum 锚点
ref_gnss_11: [lon_br, lat_br]       # 右下角
```

### 启动时覆盖（不修改源码）

也可以不改上述文件的默认值，在 launch 时用参数覆盖：

```bash
# 步骤 2 - bringup：指定 gnss_odom_map_tf 用的地图
ros2 launch workspace_ros real_boat_bringup.launch.py \
  localization_backend:=mavros_odom \
  enable_nav2_cmd_vel_to_mavros:=true \
  map_config_yaml:=/绝对路径/新地图.yaml

# 步骤 3 - Nav2：指定 map_server 载入的地图（必须与上一步同一 YAML）
ros2 launch workspace_nav nav2_real_mavros.launch.py \
  map:=/绝对路径/新地图.yaml
```

### 校验清单

- [ ] 新地图 YAML 的 `ref_gnss_10` **经纬度已实际测绘**（不能用旧图的角点坐标）
- [ ] `navsat.yaml` datum **已改为** `[ref_gnss_10.lat, ref_gnss_10.lon, 0.0]`
- [ ] `gnss_odom_map_tf` 读取的 `map_config_yaml` 与 `map_server` 加载的是**同一份 YAML**
- [ ] `waypoint_transform` 的 `map_yaml_path`（或 fallback）指向同一 YAML
- [ ] PX4 HOME 设为 `ref_gnss_10` 对应经纬度：`ros2 service call /mavros/cmd/set_home mavros_msgs/srv/CommandHome "{current_gps: false, latitude: XX, longitude: XX, altitude: 0.0}"`
- [ ] RViz 中 `/map` 与 `global_costmap/costmap` 几何无拧转
- [ ] 地面站下发的航点经纬度落在地图覆盖范围内

### 参数对应关系图

```
map.yaml (Nav2 map_server 加载)
  └── ref_gnss_10 [lon, lat]
        │
        ├──► navsat.yaml datum ───► EKF navsat_transform（GPS→odom）
        ├──► gnss_odom_map_tf ───► map→odom 动态变换
        ├──► waypoint_transform ───► 经纬度→地图 x,y
        └──► PX4 HOME ───────────► /mavros/local_position/odom 局域原点
```

**核心原则：上面 4 条链路必须共享同一个 `ref_gnss_10`，即共享同一份 map.yaml。**
